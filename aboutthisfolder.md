## Why package.json is in the Root Directory

This Python project includes a `package.json` file in the root directory for the following reasons:

### Model Context Protocol (MCP) Server Dependencies

- **Primary Purpose**: Manages Node.js-based MCP servers that extend VS Code functionality
- **Key Dependencies**:
  - `@brightdata/mcp` - Web scraping and data extraction capabilities
  - `task-master-ai` - Task management and project organization tools

### Integration Requirements

- **VS Code MCP Integration**: VS Code's MCP configuration expects Node.js dependencies in the project root
- **Script Execution**: Shell scripts in `/scripts/` use `npx` commands that must resolve from the root
- **Development Consistency**: Ensures all team members have the same MCP server versions

### Installation Requirements

```bash
# Install Python dependencies (primary application)
poetry install --with dev

# Install Node.js dependencies (MCP servers)
npm install
```

**Note**: While the core application is Python-based, this hybrid approach leverages the best tools from both Python and Node.js ecosystems through the Model Context Protocol.
